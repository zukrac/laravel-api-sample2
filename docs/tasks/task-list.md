**Тестовое задание "Создание REST API приложения"**

**Описание.**

Необходимо реализовать **REST** **API** приложения для справочника _Организаци__й_, _Зданий_, _Деятельност__и_.

1. [x] **Организация** - Представляет собой карточку организации в справочнике и должна содержать в себе следующую информацию:
	1. [x] Название: Например ООО “Рога и Копыта”
	2. [x] Номер телефона: организация может иметь несколько номеров телефонов (2-222-222, 3-333-333, 8-923-666-13-13)
	3. [x] Адрес: Организация должна находится в одном конкретном здании (Например, Блюхера, 32/1)
	4. [x] Деятельность: Организация может заниматься несколькими видами деятельностей (Например, “Молочная продукция”, “Мясная продукция”)
    

2. [x] **Адрес** - Содержит в себе как минимум информацию о конкретном здании, а именно:
	1. [x] адрес
	2. [x] координаты

3. [x] **Категория** - позволяет классифицировать род деятельности организаций в каталоге. Имеет название и может в древовидном виде вкладываться друг в друга. Пример возможного дерева деятельности:
    

  - Еда
	  - Мясная продукция
	  - Молочная продукция
  - Автомобили
	  - Грузовые
	  - Легковые
		  - Запчасти
		  - Аксессуары

**Функционал приложения.**

Взаимодействие с пользователем происходит посредством **HTTP** запросов к **API серверу** с использованием **статического API ключа**. Все ответы должны быть в формате **JSON**. Необходимо реализовать следующие методы:
- companies-by-address
- companies-by-category
- companies-by-location    
- company/get
- company/search
	- address
	- category
	- location
	- category-list
	- name
    
- поиск организации по названию
    
- ограничить уровень вложенности деятельностей 3 уровням

	- [x] категории
	- [x] компании
	- [x] телефоны
	- [x] адреса
